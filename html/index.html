ビデオ背景除去アプリ: 研究レポート

インフォグラフィック

研究概要

この研究は、PythonのFlaskフレームワーク、FFmpeg、そしてAIベースのrembgライブラリ、特にBiRefNet-Portraitモデルを使用してビデオ背景除去ウェブアプリケーションを構築する詳細を記しています。主要な目的は、動画のアップロードから背景除去、そして透明な背景を持つ新しい動画への再エンコードまでを効率的に処理し、非同期処理を活用することです。GPUアクセラレーションとBiRefNet-Portraitモデルの組み合わせにより、高い精度が達成されます。

透明度をサポートするアルファチャンネルはWebM（VP9）やMOV（ProRes 4444）のようなフォーマットで利用可能ですが、Redditのようなプラットフォームでは透明領域が黒く表示される場合があることが指摘されています。この研究では、Cloudflareの画像セグメンテーションに関する取り組み、rembgのGitHubリポジトリ、ビデオエディタの互換性に関するRedditの議論、そしてUnscreen（現在はCanvaの一部）やVideoBGRemover.comといった商用サービスも参照し、AI背景除去市場と技術トレンドに関する文脈を提供しています。

技術実装詳細と補足

rembgとBiRefNet-Portrait、GPUアクセラレーション

rembgをbirefnet-portraitモデルとGPUアクセラレーションで利用するには：

  1. インストール： GPUサポート付きでrembgをインストールします。pip install "rembg[gpu]"を実行します。これにより、CUDA対応アクセラレーションのためのonnxruntime-gpuが含まれます。
  2. Pythonでの使用：
    • rembgからremoveとnew_sessionをインポートします。
    • モデルを指定してセッションを作成します：session = new_session("birefnet-portrait")。
    • 画像（バイトとして読み込まれたもの）を処理します：output_bytes = remove(input_bytes, session=session)。
    • remove関数は、onnxruntime-gpuが正しくインストールされ設定されていれば、自動的にGPUを活用します。

FFmpegでのビデオ透明度（アルファチャンネル）の処理

FFmpegは、アルファチャンネルの処理を含むビデオ操作に使用されます。

  • アルファチャンネル付きMOV：
    • ProRes 4444： prores_ksエンコーダを-profile:v 4444およびyuva444p10leのようなピクセルフォーマットで使用します。
    • QuickTime Animation (QTRLE)： qtrleエンコーダをargbのようなピクセルフォーマットで使用します。
  • アルファチャンネル付きWebM：
    • libvpx-vp9コーデックを使用します。
    • 一般的にyuva420pピクセルフォーマットを使用します。
    • 重要な点として、既存のアルファ付きWebMファイルをデコードする際には、正しいアルファデコードを保証するために、入力ファイル（-i）の前に-c:v libvpx-vp9を指定します。
  • Python subprocessモジュール： FFmpegコマンドは、引数のリストとしてコマンドを渡すことでsubprocess.run()を使用して実行できます。

Celeryによる非同期タスク処理

ウェブアプリケーションをブロックすることなく、時間のかかるビデオ処理タスクを管理するためには：

  • Celery： 分散タスクキューシステムです。
  • メッセージブローカー： FlaskとCeleryワーカー間の通信に必要です（例：RabbitMQまたはRedis）。
  • ワークフロー：
    1. Flaskがビデオアップロードを受け取ります。
    2. Flaskは.delay()を使用してCeleryに処理タスクをディスパッチします。
    3. Flaskはユーザーに即座にレスポンスを返します（例：タスクID付き）。
    4. Celeryワーカーがタスクをピックアップします。
    5. ワーカーはFFmpegによるフレーム抽出、rembgによる背景除去、FFmpegによる再エンコードを実行します。
    6. タスクのステータスと進捗が更新されます。
    7. フロントエンドはステータス更新のためにFlaskをポーリングし、最終的なビデオリンクを取得します。

  • AIモデル比較： コンパクトな表またはグラフでAIモデルの比較結果を表示し、GPU使用時のbirefnet-portraitの高い精度を強調。

  • ウェブアプリケーション（Flask）： 左側に配置され、ユーザーインターフェース、ファイルアップロード、処理ステータス表示の機能をフローチャートで説明。
